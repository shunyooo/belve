# Plan Compliance Review

実装がプランの意図通りに完了しているかをチェックする。

## レビュー手順

1. **プランの確認**: 会話コンテキストからプラン（Plan モードで作成されたもの、またはユーザーが提示した要件）を特定する
2. **変更差分の取得**: `git diff` でステージ済み・未ステージの全変更を確認する
3. **適合性チェック**: 以下の観点で各変更をプランと照合する

## チェック項目

### 完全性
- [ ] プランで挙げた全ステップが実装されているか
- [ ] 未着手のステップはないか
- [ ] 各ステップの受け入れ条件を満たしているか

### 正確性
- [ ] プランの意図から逸脱した実装はないか
- [ ] プランにない余計な変更（スコープクリープ）が含まれていないか
- [ ] 変更がプランで指定したファイル・モジュールに閉じているか

### 整合性
- [ ] 変更間に矛盾はないか（あるファイルで追加した機能が別ファイルで未接続等）
- [ ] import / 依存関係は正しく繋がっているか
- [ ] 既存コードとの結合点は正しいか

## 出力フォーマット

```
## Plan Review Result

### Status: PASS / FAIL / WARN

### Completed Steps
- Step 1: [説明] -- OK
- Step 2: [説明] -- OK

### Issues Found
- **[MISSING]** Step N が未実装: [詳細]
- **[DEVIATION]** file.swift:42 — プランでは X だが Y として実装されている
- **[SCOPE_CREEP]** file.swift:15 — プランにない変更: [詳細]

### Recommendation
[次に取るべきアクション]
```
